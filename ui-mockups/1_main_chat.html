根据最新的系统架构设计（v3.2）：

当前的 BGE-Reranker-Base 模型是通过 ProcessPoolExecutor 放入独立的 CPU 进程池中进行推理的。

这样设计的目的是为了避免纯 CPU 计算阻塞 FastAPI 的单线程异步事件循环（asyncio）。如果不做隔离，单次 Reranker 推理（约100ms）会挂起整个主线程，导致高并发场景下其他用户请求（甚至包括简单的静态资源获取）被完全阻塞，从而引发严重的性能瓶颈 1。
